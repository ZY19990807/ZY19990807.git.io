Hexo Github个人网站搭建教程

前言:在网上找了许多博客网站，觉得东西还是自己的好。1月份就搭建好了该网站，这几天为了找当初搭建网站时出的错误，全部删掉又重新来过，途中还是跌跌撞撞的，参考了许多大佬的教程和经验分享，为做好记忆备份，特写此文章。

操作系统：Window10

Hexo+Github个人网站搭建的几大步骤
1.前期准备
安装Git
安装Node.js
注册Github账号
购买域名
2.安装Hexo (完成此步骤你可以查看到：localhost：端口号 页面)
3.推送网站，即链接Hexo和Github(完成此步骤你可以查看：你的仓库名.github.io)
新建仓库
选择主题
链接Hexo和Github
4.绑定域名（完成此步骤你可以直接通过自己的域名查看网站）

进入右键 Git Bash Here

注：参考了很多教程，这个步骤会有些许不同。可以像我一样先新建文件夹，安装Hexo到该文件夹中。也可以直接在终端下先安装Hexo，再通过Hexo init <文件夹> 初始化命令，自动生成新文件夹。如果按照后一种方法，在上一条初始化命令后需要cd <文件夹> 再进行下一步npm install

在弹出的命令行窗中键入
    npm install hexo-cli -g
运行该命令可能会报错，不影响后续操作。速度较慢，请耐心等待。 
也可以安装国内镜像进行加速

    hexo -v
查看Hexo版本，说明安装成功
    hexo init 
初始化文件夹
    npm install 
安装必要的组件
    hexo g
生成静态网页
    hexo s
打开本地服务器
在浏览器中输入：localhost:4000 如图所示
ctrl+c关闭本地服务器

注：
如果localhost:4000打不开请尝试如下操作：

Hexo 3.0 把服务器独立成了个别模块，您必须先安装 hexo-server 才能使用。
    npm install hexo-server --save
安装完成后，输入以下命令以启动服务器，您的网站会在 http://localhost:4000 下启动。在服务器启动期间，Hexo 会监视文件变动并自动更新，您无须重启服务器。
 hexo server

如果您想要更改端口，或是在执行时遇到了 EADDRINUSE 错误，可以在执行时使用 -p 选项指定其他端口，如下：
 hexo server -p 5000

卸载Hexo
3.0.0版本执行
npm uninstall hexo-cli -g
之前版本执行
npm uninstall hexo -g

推送网站
新建仓库
登录Github账户，新建仓库
https://github.com/new
仓库名后必须加.github.io，页面最下的README初始化也要勾上
选择主题
仓库中点击settings,拉到最下Github Pages,点击Choose a theme，选择好之后回到Github Pages。Github Pages会提示Your site is ready to be published at 链接。点击链接你就可以看到所选的样式。

链接Hexo和Github
    在<blog>中，打开Git Bash,输入以下命令
        git config --global user.name "你的GitHub用户名" ，回车
        git config --global user.email "你的GitHub注册邮箱"  ，回车
        ssh-keygen -t rsa -C "你的GitHub注册邮箱"，三个回车
    命令行会提示你生成的文件id_rsa.pub在哪里，根据路径找到该文件后用记事本打开，复制所有内容。
    打开Github，打开settings,找到SSH and GPG keys,新建SSH heys，Title任意，Key中粘贴复制的文字，保存。如图所示。
    回到Git Bash,输入
        ssh git@github.com 或者 ssh -T git@github.com
    出现你的用户名就说明成功了。

    打开<blog>下的_config.yml文件，这是博客的配置文件，在这里你可以修改与博客相关的各种信息。
    修改最后一行的配置：
    特别注意！！！type，repo，branch前两个空格，“：”后面一个空格，差一个都不行的：
    deploy:
      type: git
      repository: url 如果 git@github.com:ZY19990807/ZY19990807.github.io
      branch: master
    保存站点配置文件。

    其实就是给Hexo d 这个命令做相应的配置，让Hexo知道你要把blog部署在哪个位置，很显然，我们部署在我们GitHub的仓库里。最后安装Git部署插件，输入命令：
        npm install hexo-deployer-git --save
    输入以下命令
        hexo clean 
        hexo g 
        hexo d 

    注：
    如果运行hexo d 
    报错ERROR Deployer not found: git
    这是因为没安装hexo-deployer-git插件，在站点目录下输入下面的插件安装就好了：
    npm install hexo-deployer-git --save
    也有可能是_config.yml中url写错了

来源：https://blog.csdn.net/qq_21808961/article/details/84476504

注：
_config.yml中的仓库地址可以写成
http://
也可以携程
git@github.com:你的Github用户名/你的仓库名.github.io

至此，在浏览器中输入 你的仓库名.github.io，效果如图

绑定域名
域名解析
添加域名解析如下图:

最开始参考的教程都是三条解析，但是第二次添加解析后出现网页重定向问题。原因未知，后改为两条解析问题成功解决。

打开Github，进入settings,拉到下面Custom domain处，填上你自己的域名，保存。
本地根目录<blog>应该显示CNAME文件，没有的话手动在source目录下，使用笔记本新建CNAME文件，写入域名，保存时没有后缀。运行以下命令:
    hexo g
    hexo d
至此你的域名已经成功绑定。  

注： 
绑定成功但显示网页重定向：

域名解析设置中的设置出现问题，应设置为
 @         A             185.199.108.153

 www      CNAME           username.github.io.

另一些文章上写到设置为如下
 @         A             192.30.252.153

 @         A             192.30.252.154

 www      CNAME           username.github.io.

You must update your site's DNS records if you'd like it to be available via your custom domain. For more information, see
